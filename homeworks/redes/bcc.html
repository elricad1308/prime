BCC

This section provides a basic overview or the BCC protocol.

What it is?

BCC (Blocked Customer Cleared) is a protocol used in network systems. Generally, it consists of a set of n servers that can attend n clients. The state i indicates that i servers are attending i clients.

In BCC, clients can arrive at any time. At each instant in time, there is a probability λ that a new client arrives to the system. The probability that two or more clients arrive at the exact same instant is so small that is effectively zero. Clients are attended by servers, and with probability μ a client will be attended at a given instant (and therefore, it leaves the system). When n clients are inside the system and another client arrives, the system is said to be blocked. The plot above shows the probability that this event occurs.

How do I read the plot?

The probabilities λ and μ are show in the x & y axis. The z axis represents the blocking probability, that is, the probability that a client arrives and it cannot be attended.

Three layers of data are present on the plot (you can trigger them by clicking the corresponding buttons). The first layer is the blocking probability computed using a recursive method. The second layer is an approximation obtained by a simulation of the chain. Finally, the third layer is computed by using the Gauss-Seidel method.
